Backend Overview

Green Shop is an e-commerce backend API built with Node.js and Express.js. It provides RESTful endpoints for managing products, shopping cart items, delivery options, orders, and payment summaries.

Main Purpose

To serve as the server-side component of a green-themed e-commerce web application, handling data persistence, business logic, and serving static assets (product images and frontend build).

Overall Architecture

The backend follows a layered architecture with Express.js handling HTTP requests, Sequelize ORM managing database interactions, and route modules encapsulating business logic. There is no separate controller or service layer — route handlers directly contain the business logic. The database defaults to SQLite (via sql.js-as-sqlite3) for local development, with support for MySQL or PostgreSQL when RDS environment variables are configured.

Database Layer

The backend uses Sequelize ORM with SQLite as the default database (in-memory with file persistence). For production, it supports MySQL and PostgreSQL via RDS environment variables.

Database Configuration

The Sequelize instance is configured in models/index.js. It detects RDS environment variables (RDS_HOSTNAME, RDS_USERNAME, RDS_PASSWORD) to switch between SQLite and cloud databases. When using SQLite, hooks automatically persist the in-memory database to database.sqlite after write operations.

Middleware Pipeline

The middleware is applied in the following order in server.js:

CORS JSON Body Parser Static Files (/images) API Routes Static Files (/dist) SPA Catch-all Error Handler

Middleware Details

OrderMiddlewarePurpose
1cors()Enables Cross-Origin Resource Sharing for frontend requests

3express.static('/images')Serves product images, logos, icons, and rating graphics
4API Routes (/api/*)Route handlers for products, cart, orders, etc.
5express.static('/dist')Serves the built frontend SPA
